*Win32控制器版贪吃蛇*

1.设计思路：
  a.初始化：整个贪吃蛇地图就是一组内容为空的字符二维数组（墙壁除外，墙壁填充：#)

  b.贪吃蛇的填充：贪吃蛇就是将地图中的某几个相连的字符填上其他字符（@）；

  c.贪吃蛇的方向：键盘的上、下、左、右是有其Ascall码值的，在游戏的死循环中，当有键盘敲击时（kbhit():keyboard hit）可对输入内容进行判断，用getchar（）函数可得到其值，然后判断是向哪个方向；

  d.贪吃蛇的移动：将贪吃蛇的头部坐标往方向所指不停的填充字符，蛇尾再不断的清除自身；

  e.增长机制：当蛇头移动到某一位置时，就判断这个方向上的下一个位置的字符串是否为食物所代表的字符（* 为了和蛇身进行区分），是则自身长度+1（蛇尾本环节不进行擦除即可）；

  f.碰撞机制：当蛇头移动到某一位置时，就判断这个方向上的下一个位置的字符内容，空则无事，食物则长度+1，墙壁或蛇身则游戏结束。


（3）但是不进行（4），以此实现长度+1；
